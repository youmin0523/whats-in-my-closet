# DESIGN.md — "What's in my closet" 아트디렉션

> 이 문서는 **모든 UI 작업의 계약**이다. 화면을 만들 때 항상 이 문서를 컨텍스트로 두고,
> 매 패스마다 맨 아래 **안티-AI 체크리스트**를 통과시킨다. 목표: 바이브코딩 결과물이
> 뻔한 "AI 룩"으로 수렴하지 않게 하는 것.

## 컨셉

**"내 옷을 전시하는 편집숍 / 갤러리."** 삼성 비스포크 냉장고가 "안을 들여다보게" 하듯,
옷장을 차분하고 정제된 쇼룸처럼 보여준다. **UI는 중립적이고 절제**되게 — 색·생기는
콘텐츠(배경 제거된 옷 사진)가 낸다. UI가 화려하면 옷이 죽는다.

레퍼런스 북극성: 29CM · W컨셉 · 무신사 · SSENSE · Aesop의 절제미 + Linear/Arc 수준의
크래프트 + 미술관 라벨링.

## 컬러 (토큰은 `apps/web/src/app/globals.css`)

웜 뉴트럴 캔버스 + 단일 악센트(**올리브**). 기본 슬레이트/징크 ❌, AI 보라/인디고
그라데이션 ❌. 채도는 낮게 — 옷이 주인공.

| 역할 | 라이트 | 의미 |
|---|---|---|
| `background` | warm ivory (oklch 0.985) | 캔버스 |
| `foreground` | warm ink (oklch 0.22) | 본문 |
| `muted-foreground` | warm gray (oklch 0.50) | 보조 텍스트 |
| `border` | hairline (oklch 0.90) | 1px 헤어라인 |
| `primary` | **olive (oklch 0.47 0.045 135)** | 단일 악센트 |
| `secondary` | warm sand | 잔잔한 배경 |
| `destructive` | muted clay-red | 경고 |

- 악센트는 **아껴 쓴다.** CTA·활성 상태·중요 수치에만. 모든 곳에 칠하지 않는다.
- 의류 색을 위해 UI 뒤판은 항상 ivory/white. 썸네일은 배경 제거 후 흰 캔버스 위.

## 타이포

- 본문·UI: **Pretendard Variable** (자체 호스팅, `--font-pretendard`). 국문 최적.
- 별도 디스플레이 페어링 없이 **굵기·크기·자간**으로 위계를 만든다(국·영 일관성 유지).
- 큰 제목: tracking 살짝 타이트(`tracking-tight`), 무게 600–700. 본문: 400–450.
- 숫자(수량·켤레·점수)는 `tabular-nums` 또는 `slashed-zero` 고려.
- 이모지 ❌ (제품 크롬). 아이콘은 lucide 라인 아이콘을 절제 사용.

## 레이아웃 · 간격

- **중앙정렬 히어로 + 카드 3개 템플릿 ❌.** 에디토리얼 그리드 + 넉넉한 여백 +
  큰 이미지 + 의도된 비대칭.
- 옷장은 masonry/갤러리 그리드. 카드 간격은 촘촘하지 않게(숨 쉬는 여백).
- radius: `--radius` 0.625rem 기준(과하지 않게). 하드 섀도 ❌ → **헤어라인 보더 +
  아주 옅은 소프트 섀도**.
- 컨테이너 max-width는 콘텐츠에 맞춰(전폭 갤러리는 넓게, 폼은 좁게).

## 모션 (motion / framer-motion)

- **물리적이고 미묘하게.** 서랍 열림, 아이템 들림(hover lift), 진입 페이드+슬라이드(8–12px).
- duration 150–280ms, ease-out. 과한 바운스/스프링 남발 ❌.
- 접근성: `prefers-reduced-motion` 존중.

## 컴포넌트 규칙

- shadcn/ui는 **프리미티브로만** 쓰고 토큰으로 전면 리테마. "기본 shadcn 룩" 금지.
- 버튼: 기본은 솔리드 올리브(primary) / 외곽선 / 고스트. 모서리 과하지 않게.
- 빈 상태(empty state)는 **진짜로** 디자인한다(일러스트 대신 절제된 카피 + 1 CTA).
- 마이크로카피는 자연스러운 한국어. 로봇어투/번역투 ❌. ("옷을 불러오는 중…" O)

## 이미지

- 의류 썸네일: 배경 제거본을 흰/ivory 위에. `object-contain`, 일정 패딩.
- `next/image` 사용. 외부 호스트는 `images.remotePatterns`에 등록(Cloudinary 등).
